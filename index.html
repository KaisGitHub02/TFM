Diseño, Implementación y Evaluación de un Sistema de Debate Multi-Agente basado en LLMs con Moderación IA para la Síntesis de Perspectivas

Trabajo Final de Máster
Kais Bouadi Molina
Universidad de Granada
2025

Motivación y Problema

Desafío en IA: Los sistemas actuales de IA argumentativa no logran mantener debates complejos y coherentes.
Perspectivas diversas: Falta capacidad para integrar y sintetizar opiniones contrapuestas de manera efectiva.
Moderación inteligente: Escasez de mecanismos automatizados para gestionar y equilibrar la discusión.
Toma de decisiones: Oportunidad de mejorar cómo los grupos alcanzan consensos informados mediante IA.

Problema Principal

Los modelos de lenguaje actuales (LLMs) presentan limitaciones a la hora de generar argumentos equilibrados y considerar múltiples perspectivas de forma estructurada y transparente, dificultando la síntesis de conclusiones útiles en debates complejos.

Objetivos del Trabajo

Objetivo General

Diseñar, implementar y validar un sistema multi-agente capaz de simular debates estructurados, incorporando moderación automática basada en IA para la generación de síntesis equilibradas y fundamentadas de diferentes perspectivas.

Objetivos Específicos

Diseñar la arquitectura funcional de los agentes Pro, Contra, Neutral y Moderador.
Implementar algoritmos avanzados de toma de decisiones grupales (GDM) y agregación de argumentos.
Desarrollar extracción y Deduplicación semántica, eliminando duplicados y ruido
Generar resúmenes neutrales que recojan los puntos y débiles expuestos durante el debate

Estado del Arte

Principales Enfoques Actuales

Sistemas de debate automatizado: Plataformas que simulan discusiones, pero con reglas y capacidades limitadas.
Arquitecturas multi-agente en PLN: Modelos que distribuyen roles entre agentes para tareas de procesamiento de lenguaje natural.
LLMs para argumentación: Utilización de modelos de lenguaje generativo para producir argumentos, con resultados desiguales.
Moderación de contenido: Herramientas automáticas para filtrar o guiar debates, generalmente centradas en moderación superficial.

Limitaciones Detectadas

Moderación poco sofisticada: Falta de mecanismos inteligentes para guiar y equilibrar el debate.
Sesgo en los argumentos: Tendencia de los LLMs a favorecer ciertos puntos de vista o repetir patrones.
Ausencia de síntesis balanceada: Dificultad para integrar múltiples perspectivas de forma objetiva y estructurada.
Evaluación subjetiva: La calidad argumentativa suele medirse de forma manual y poco reproducible.

Arquitectura del Sistema Multi-Agente

Agente Pro

Rol: Defensor de la posición afirmativa

Función: Genera argumentos sólidos a favor del tema

Agente Contra

Rol: Crítico de la posición afirmativa

Función: Desarrolla contra-argumentos fundamentados

Agente Neutral

Rol: Mediador imparcial

Función: Identifica puntos de convergencia y matices

Moderador IA

Rol: Coordinador inteligente

Función: Analiza, evalúa y sintetiza perspectivas

Flujo de Procesamiento

1. Generación paralela: Los tres agentes especializados procesan el tema simultáneamente

2. Síntesis inteligente: Integración de perspectivas usando algoritmos GDM y OWA

3. Análisis semántico: El Moderador IA evalúa coherencia y calidad argumentativa

Implementación Técnica

Componentes Técnicos Implementados

Generación Multi-Agente

Prompts especializados para cada rol (Pro, Contra, Neutral) con LLMs via API

Deduplicación Semántica

Eliminación de argumentos redundantes usando Sentence-Transformers y similitud coseno

Síntesis Inteligente

Moderador IA que analiza y consolida perspectivas múltiples en respuesta unificada

Evaluación Automática

Métricas de calidad, coherencia y diversidad usando modelos de embedding

Métricas de Evaluación Implementadas

Coherencia: Análisis semántico de consistencia interna
Diversidad: Medición de variabilidad entre perspectivas de agentes
Calidad de síntesis: Evaluación de la integración efectiva de argumentos
Equilibrio: Representación balanceada de todas las perspectivas

Stack Tecnológico

Backend: Python, Hugging Face Transformers, Sentence-Transformers

Frontend: Streamlit para interfaz web interactiva

APIs: Integración con LLMs via API (compatible con Google Colab gratuito)

Procesamiento: NumPy, scikit-learn y embeddings para análisis semántico y métricas

Limitaciones y Trabajo Futuro

Limitaciones Identificadas

Dependencia de LLMs base: Calidad limitada por las capacidades del modelo subyacente
Gestión de contexto: Escalabilidad limitada para debates prolongados sin frameworks como LangChain
Costos computacionales: Tiempo de procesamiento superior a enfoques directos
Adaptación de dominio: Necesidad de ajuste fino para temas muy especializados
